# Buddy Reroller 界面设计规格

> 文档角色：完整描述当前 Web UI 的页面结构、视觉设计、功能交互和交互表现
> 事实源：代码（frontend/ 目录）
> 最后更新：2026-04-04 | 操作人：lyk

## 1. 全局设计语言

### 1.1 主题风格

"Cozy Terminal" — 暖琥珀色深色主题，像素游戏宠物收集 × 温暖终端美学。刻意避免 AI 仪表盘常见的冷蓝/灰色调。

### 1.2 色彩体系

| 用途 | 变量 | 色值 | 说明 |
|------|------|------|------|
| 最深背景 | `--bg` | `#0d0c0a` | 近纯黑暖色 |
| 表面层级 0 | `--surface-0` | `#16140f` | 卡片背景 |
| 表面层级 1 | `--surface-1` | `#1c1a14` | 输入框、子面板 |
| 表面层级 2 | `--surface-2` | `#242119` | 进度条底色、标签 |
| 表面层级 3 | `--surface-3` | `#332e26` | 最高层级表面 |
| 主强调色 | `--accent` | `#e8a849` | 琥珀金 |
| 强调色悬停 | `--accent-hover` | `#f0ba60` | 亮琥珀 |
| 正文色 | `--text` | `#ece8df` | 暖白 |
| 次要文字 | `--text-secondary` | `#b0a898` | 暖灰 |
| 弱化文字 | `--text-dim` | `#7a7265` | 深暖灰 |
| 边框 | `--border` | `#3d382e` | 暖褐 |

**稀有度色彩**：
| 稀有度 | 色值 | 用途 |
|--------|------|------|
| Legendary | `#ffc93c` 金色 | 标签、卡片左边框、发光效果 |
| Epic | `#c792ea` 紫色 | 标签、卡片左边框 |
| Rare | `#82aaff` 蓝色 | 标签、卡片左边框 |
| Uncommon | `#7ec699` 绿色 | 标签、卡片左边框 |
| Common | `#7a7265` 灰色 | 标签、卡片左边框 |

**属性色彩**：
| 属性 | 色值 |
|------|------|
| DEBUGGING | `#3b82f6` 蓝 |
| PATIENCE | `#22c55e` 绿 |
| CHAOS | `#ef4444` 红 |
| WISDOM | `#a855f7` 紫 |
| SNARK | `#f59e0b` 琥珀 |

### 1.3 字体

| 用途 | 字体 | 来源 |
|------|------|------|
| 标题 | Space Grotesk | Google Fonts |
| 正文 | Nunito Sans (400/500/600/700) | Google Fonts |
| 代码/种子/许可证 | JetBrains Mono | Google Fonts |

### 1.4 圆角与阴影

| 层级 | 圆角 | 用途 |
|------|------|------|
| `--radius-lg` | 14px | 大卡片 |
| `--radius` | 10px | 标准卡片 |
| `--radius-sm` | 7px | 按钮、输入框 |
| `--radius-xs` | 5px | 标签、徽章 |

阴影统一使用暖色调（`rgba(20, 18, 12, ...)`），三个层级：sm/md/lg。

### 1.5 背景纹理

Body 上层叠加一个 SVG 噪点纹理（fractalNoise），2.5% 透明度，`pointer-events: none`，固定定位。为深色背景增加微妙的材质感。

## 2. 页面整体布局

```
┌─────────────────────────────────────────────────────┐
│  Header（标题 + 副标题 + PRO 徽章 + 语言切换）       │
│  ─── 渐变分隔线 ───                                  │
├─────────────────────────────────────────────────────┤
│  [环境不兼容警告] （条件显示）                        │
│  [当前伙伴卡片] （条件显示）                          │
├─────────────────────────────────────────────────────┤
│  ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐       │
│  │ 🔍 搜索 │ │ 📦 收藏 │ │ 📖 图鉴 │ │ ⚙️ 设置 │   │
│  └────────┘ └────────┘ └────────┘ └────────┘       │
│  标签栏（Segmented Control 风格）                     │

│  max-width: 1100px, 居中                              │

- 容器最大宽度 1100px，水平居中，两侧 padding 32px

- **搜索标签页**使用 `display: none` 切换（非条件渲染），保持 WebSocket 连接不中断

## 3. Header 区域

### 3.1 结构

```
                  🐥 Buddy Reroller [PRO]
        Find your perfect Claude Code companion

        Unlock PRO: auto-pick highest stats → Get Pro ←
                            [EN/中文]
```

### 3.2 元素说明

| 元素 | 说明 | 条件 |
|------|------|------|
| 标题 | "Buddy Reroller"，Space Grotesk 字体 | 始终显示 |
| PRO 徽章 | 金色渐变背景 + 发光阴影，"PRO" 字样 | 仅在 Pro 激活时 |
| 副标题 | 描述文字 | 始终显示 |
| Pro 推广文案 | 含购买链接的 HTML 文本 | 仅在非 Pro 时 |
| 语言切换 | 右上角圆角药丸按钮，显示 "EN" 或 "中文" | 始终显示 |
| 分隔线 | 底部渐变线（透明 → 边框色 → 透明） | 始终显示 |

### 3.3 交互

- **语言切换**：点击在 en/zh 之间切换，立即生效，持久化到 localStorage
- **Pro 推广链接**：点击新标签页打开爱发电购买页

### 5.2 搜索进度区

搜索启动后，进度区动态展示：

#### Phase 1：种子匹配

```
  搜索中... 4,294,967,296 次尝试 · 88.7s · 48,408,179/s · 100.0% [4x 并行]
  ████████████████████████████████████████░░░░░░░░░░░░░░░░░░░░
```

- 文字行：状态文本 + 尝试次数 + 耗时 + 速度 + 百分比 + "4x 并行" 徽章
- 进度条：琥珀色渐变填充，平滑宽度过渡动画
- 每 500ms 更新一次

#### Phase 2：反向解析 userID

```
  ✅ 已匹配种子                                  [4x 并行]
  解析 userID… 1,234,567 次反查 · 12.3s · 100,234/s
  ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓
  正在从 43 亿个 ID 中反向匹配 — 通常 1~3 分钟，请耐心等待
```

- 第一行：✅ 标记 Phase 1 完成 + 并行徽章
- 第二行：Phase 2 独立的尝试数 + 耗时 + 速度
- 进度条：脉冲条纹动画（indeterminate），视觉上明确表示仍在工作
- 底部提示：说明当前操作和预计时间

#### 搜索未找到

当整个种子空间遍历完毕仍未匹配时，显示琥珀色警告框。

#### 错误

红色边框提示框，显示连接错误等服务端错误。

### 5.3 搜索结果区

找到伴侣后显示，带淡入 + 上滑庆祝动画（0.4s ease-out）。

```
┌─ 🔍 找到啦！共尝试 1,234 次（5.6秒）！ ─────────────┐
│  （或 ⚡ 数据库命中！）                                │
│                                                       │
│  候选数: 1 · 总属性: 425                              │
│                                                       │
│  [⚠️ userID 反查超时提示] （仅在 fallback 时显示）     │
│                                                       │
│  ┌─ 伴侣展示 ─────────────────────────────────────┐  │
│  │  见第 8 节 CompanionDisplay                      │  │
│  └─────────────────────────────────────────────────┘  │
└───────────────────────────────────────────────────────┘
```

- 标题区分数据库命中（⚡）和实时搜索（🔍）
- 候选数和总属性值显示
- Fallback 提示：仅在 Phase 2 超时时显示（琥珀色警告框）

### 5.4 搜索流程图

```
用户点击"开始搜索"
    │
    ├── Step 1: POST /api/search/lookup（数据库预查）
    │   ├── 命中 → 直接显示结果（source: "database"）
    │   └── 未命中 ↓
    │
    └── Step 2: WebSocket 实时搜索
        │
        ├── 连接 ws://${host}/ws/search
        │   └── 发送 { type: "start", criteria, peakStat, dumpStat }
        │
        ├── Phase 1: 种子匹配（多 Worker 并行）
        │   ├── 接收 progress 消息 → 更新进度条
        │   └── 找到匹配种子 → 进入 Phase 2
        │
        ├── Phase 2: 反向解析 userID
        │   ├── 接收 progress 消息（含 phase2Attempts）
        │   ├── 找到 userID → 发送 found 消息
        │   └── 超时（10 分钟）→ 发送 found + fallback: true
        │
        └── 用户点击"停止搜索"
            └── 发送 { type: "stop" }
```

### 5.5 搜索后的操作

找到伴侣后，CompanionDisplay 组件显示两个操作按钮：

| 按钮 | 行为 | API |
|------|------|-----|
| **应用** | 将 userID 写入 `~/.claude.json`，替换当前伴侣 | `POST /api/apply` |
| **应用（fallback）** | 保存到收藏集（无真实 userID） | `POST /api/collection/add` |
| **收藏** | 保存到本地收藏集 | `POST /api/collection/add` |

操作成功后弹出 alert 提示。

└──────────────────────────────────────────────────┘
```

### 9.2 精灵图

- 使用 `renderSprite(species, hat)` 生成 ASCII 艺术帧
- 渲染在 `<pre>` 标签内，等宽字体，10px 字号
- 精灵框有暗色背景和边框

### 9.3 稀有度标签

彩色药丸标签，每种稀有度对应独立颜色和背景。闪光标签为金色 + "✨ SHINY"。

### 9.4 属性条

- 5 条水平属性条（DEBUGGING, PATIENCE, CHAOS, WISDOM, SNARK）
- 每条包含：3 字母缩写 + 填充条 + 数值
- 填充条颜色按属性类型独立着色
- 巅峰属性条：绿色渐变填充，绿色标签
- 短板属性条：红色渐变填充，红色标签
- 填充动画：`bar-fill` 关键帧动画，0.6s ease-out，使用 CSS 变量 `--target-width`

### 9.5 操作按钮

| 按钮 | 样式 | 行为 |
|------|------|------|
| 应用 | 绿色 `btn-success` | 调用 `onApply` 回调 |
| 收藏 | 轮廓 `btn-outline` | 调用 `onCollect` 回调 |

- **English (en)** — 默认

- 收藏面板（6）
- 图鉴面板（9）

## 15. WebSocket 消息协议

### 客户端 → 服务端

| 类型 | 字段 | 说明 |
|------|------|------|
| `start` | criteria, peakStat, dumpStat | 开始搜索 |
| `stop` | — | 停止搜索 |

### 服务端 → 客户端

| 类型 | 字段 | 说明 |
|------|------|------|
| `progress` | phase, attempts, elapsed, parallel, progressPct, phase2Attempts | 进度更新（每 500ms） |
| `found` | userID, companion, totalStats, candidatesFound, attempts, elapsed, fallback, seed, source | 搜索成功 |
| `not_found` | attempts | 种子空间耗尽 |
| `error` | message | 服务端错误 |
| `stopped` | attempts | 用户主动停止 |
